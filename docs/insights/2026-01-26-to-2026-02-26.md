## 摩擦点分析

### 1. 过度规划与过度工程

**问题**: Claude 频繁在您希望直接执行时陷入探索和规划阶段。

**解决方案**: 更明确地表达希望立即执行而非详细计划或代理委托。

**示例**: Claude 进入规划模式并创建大量探索代理和详细计划，而不是直接创建请求的 Excalidraw 文件。

### 2. 实现细节不一致

**问题**: Claude 做出的假设与您的需求不匹配。

**解决方案**: 前期明确指定确切要求，特别是语言选择和署名格式。

**示例**:
- Claude 建议保留 Claude 和 GLM 作为共同作者，而您只想要 GLM-4.7
- Claude 最初创建英文文档，而您想要中文

### 3. 插件和技能安装失败

**问题**: 尝试安装或使用插件/技能时反复遇到问题。

**解决方案**: 请求安装前验证插件可用性，考虑手动变通方法。

**示例**: 多次尝试安装 vuejs-ai/skills 插件失败，最终在多次中断后切换到 git-workflow 技能。

```

### 2. 提交前验证
**建议**: 在假设完成之前在浏览器中测试。

```
在实现认证功能后，启动开发服务器并验证完整的认证流程（登录、登出、受保护路由）后再提交更改。
```

### 3. 插件可用性检查
**建议**: 在尝试安装之前检查插件是否存在。

```
在安装任何 marketplace 插件或技能之前，首先搜索或验证其是否存在："在尝试安装之前检查 [plugin-name] 在 marketplace 中是否可用。"
```

### 4. 明确的文件创建请求
**建议**: 请求直接文件创建而不是规划模式。

```
跳过大量规划和代理探索。直接创建文件并根据反馈迭代。
